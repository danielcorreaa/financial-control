MongoDB Atlas

Banco de dados gratuito na nuvem (512 MB)

Gratuito
  1. 1

    Acesse mongodb.com/atlas e crie uma conta gratuita. Clique em Create Free Cluster e escolha o plano M0 Free.

  2. 2

    No menu lateral, vá em Database Access → Add New Database User. Crie um usuário com senha forte. Anote as credenciais.

  3. 3

    Em Network Access → Add IP Address, adicione 0.0.0.0/0 para liberar o acesso do Railway.

  4. 4

    Clique em Connect → Drivers, copie a connection string e substitua <password> pela senha criada no passo 2:

Connection String 
mongodb+srv://usuario:senha@cluster0.xxxxx.mongodb.net/financial_control?retryWrites=true&w=majority
⚠️ Guarde esta string em segredo. Ela contém seu usuário e senha do banco.
2

Backend no Railway

Deploy do Spring Boot com Docker

Railway
  1. 1

    Acesse railway.app e faça login com sua conta do GitHub.

  2. 2

    Clique em New Project → Deploy from GitHub repo e selecione o repositório financial-control.

  3. 3

    Em Settings → Source, o Root Directory deve estar vazio (o Dockerfile está na raiz do projeto).

  4. 4

    Vá em Variables e adicione as variáveis abaixo:

Variável Valor
MONGODB_URI Connection string do Atlas (passo 1) ⬅ obrigatório
JWT_SECRET String aleatória longa (mínimo 32 caracteres) ⬅ obrigatório
ALLOWED_ORIGINS URL do Vercel — preencher após o passo 3 ⬅ volte aqui depois
💡 Para gerar um JWT_SECRET seguro, abra o terminal e rode: node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
  1. 5

    Aguarde o build (≈3 min). Quando aparecer ✓ Deployed, copie a URL gerada. Exemplo:

URL do backend 
https://financial-control-production.up.railway.app
3

Frontend no Vercel

Deploy do React/Vite — CDN global gratuito

Vercel
  1. 1

    Acesse vercel.com, faça login com GitHub e clique em New Project → Import o repositório.

  2. 2

    Em Root Directory, clique em Edit e selecione a pasta frontend.

  3. 3

    Em Environment Variables, adicione:

VariávelValor
VITE_API_URL URL do Railway do passo 2 ⬅ sem barra no final
Exemplo .env 
VITE_API_URL=https://financial-control-production.up.railway.app
  1. 4

    Clique em Deploy. Após o build, copie a URL gerada (ex: https://fincontrol.vercel.app).

  2. 5

    Volte ao Railway e atualize a variável ALLOWED_ORIGINS com a URL do Vercel. Isso libera o CORS entre frontend e backend.

✅ Teste abrindo a URL do Vercel no celular. Se o login funcionar, está tudo certo!

Ícones do App (PWA)

Necessário para "Adicionar à tela inicial"

PWA
  1. 1

    Acesse favicon.io/favicon-generator e configure:

CampoValor sugerido
TextF (ou 💰)
BackgroundRounded
Background Color#7c3aed
Font Color#ffffff
Font Size70
  1. 2

    Clique em Download. O zip contém vários tamanhos de ícone.

  2. 3

    Renomeie e copie para a pasta frontend/public/:

Arquivos necessários 
# Renomear e colocar em frontend/public/
android-chrome-192x192.png  →  icon-192.png
android-chrome-512x512.png  →  icon-512.png
favicon.ico                 →  favicon.ico
  1. 4

    Faça commit e push. O Vercel vai redeployar automaticamente com os ícones.

📱

Instalar no Celular

Depois que o deploy estiver pronto, abra a URL do Vercel no celular e adicione à tela inicial. O app vai parecer nativo — sem barra do navegador.

🤖

Android (Chrome)

  1. Abra a URL no Chrome
  2. Toque em ⋮ (menu)
  3. Toque em Adicionar à tela inicial
  4. Confirme o nome e instale
🍎

iOS (Safari)

  1. Abra a URL no Safari
  2. Toque no botão ⬆ Compartilhar
  3. Role e toque em Adicionar à Tela de Início
  4. Confirme o nome e toque em Adicionar
💡 As notificações push funcionam em Android. No iOS, só funcionam a partir do iOS 16.4 quando o app é instalado via Safari (não pelo Chrome).
